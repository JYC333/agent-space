# Capture Memory Extraction Evolution Constraints

- Raw capture is Activity first, not active Memory.
- Memory update must be proposal-first.
- Distinguish exploration, accepted_decision, stable_preference, rejected_option, and unresolved_question.
- Do not turn one-off questions into long-term memory.
- Do not bake individual personal preferences into core capability defaults.
- Conflicting or uncertain items should be marked unresolved.
- Only repeated confirmation or accepted proposal outcomes can promote content to stable memory.
- Evolution engines must output artifacts and proposals, not direct mutations.
